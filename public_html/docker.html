Docker Hub and Docker Desktop

Return to Main Page
Docker is a platform used to build and run images from
applications. Docker Desktop is a program that is used
to run those images as containers. Multiple containers with
databases can also be used to persist data on the containers.
Docker Hub is used to host the images in repositories, which
can be used with Azure to run the image as a container online.

In order run an app through Docker, a Dockerfile (with no
file extension) needs to be made. This file creates the image
that works with Docker. The following code is used in our Dockerfile: 
		
		FROM httpd:2.4
		COPY ./public_html/ /usr/local/apache2/htdocs/
A docker-compose.yml file also needs to
be created, which determines the version, type of image to use,
the port to run the image on, SQL user and passwords for MYSQL,
and much more. The following is in our compose file: 
                
                version: "3.2"
		services:
  		  apache:
    		    build: . 
    		    ports:
      		      - "8080:80"
    		    volumes:
      		      - ./public_html/:/usr/local/apache2/htdocs/

When an application is ready to be built on Docker, you use the line
docker build -t tagname .
The -t gives the specific build a tag, which you use in place of "tagname".
The dot "." at the end specifies which directory to use in the building process.
Since we use a dot, it picks the current directory you are in.
After this, we can run the image on Docker Desktop using the line:
docker run -dp 8080:80 tagname
The d and p flag run a new container for the image and map it to the port specified,
which in this case is 8080:80. The "tagname" specifies which build to run. Sure enough
Docker Desktop will then have a new container running the app you have just created.
